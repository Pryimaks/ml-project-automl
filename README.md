# Fake Job Classification: Baseline vs AutoML
Python MLflow Hydra AutoML

Цей репозиторій містить MLOps проект для класифікації типів вакансій . Основна мета — демонстрація підходу Reproducible Research з використанням локального трекінгу експериментів та автоматичного пошуку моделі.

**Важливо:** У цьому проєкті реалізовано стратегію усунення Data Leakage. Ознака `fladuent` була примусово видалена з навчальних даних, щоб уникнути тривіального передбачення. Моделі навчаються виключно на інших характеристиках вакансій.

## Структура Проєкту
Проєкт дотримується адаптованої структури **Cookiecutter Data Science**:

├── configs/ # Конфігурація параметрів (Hydra/YAML)
│ └── main.yaml # Головний конфіг (шляхи, параметри моделі)
├── data/
│ ├── raw/ # Вихідні дані (fake_job.csv)
│ └── processed/ # (Опціонально) Оброблені дані
├── mlruns/ # Локальне сховище артефактів та метрик MLflow
├── notebooks/ # Jupyter Notebooks для EDA
├── src/ # Вихідний код
│ ├── train_baseline.py # Скрипт навчання Random Forest
│ └── train_automl.py # Скрипт пошуку моделі через FLAML
├── requirements.txt # Залежності проекту
└── README.md # Документація

## Встановлення
Для запуску проекту необхідно встановити залежності. Рекомендується використовувати віртуальне середовище.

pip install -r requirements.txt

Запуск експериментів
1. Baseline Model (Random Forest)

Ручний пайплайн, надійний базовий алгоритм.

python src/train_baseline.py

2. AutoML (FLAML)

Автоматизований пошук найкращої архітектури. Обмеження часу задається у конфігу main.yaml (time_budget).

python src/train_automl.py

3. Аналіз результатів (MLflow Dashboard)

Щоб порівняти метрики, подивитися графіки (ROC Curve, Confusion Matrix) та параметри:

mlflow ui

Виберіть експеримент: fake_job_baseline
Переглядайте логи, параметри, метрики та моделі.

Результати експериментів

Порівняння ручного налаштування та AutoML. Обидва підходи показали, що датасет є достатньо роздільним для класифікації.
